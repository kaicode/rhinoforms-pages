Form declarations are contained in an array in the FormFlow.formLists map. The first Form in the 'main' formList is in bold below: 

{
  docBase: "/root",
  formLists: {
    main: [
      {
        id: "name",
        url: "/forms/example/name-form.html",
        actions: [ "next" ]
      },
      {
        id: "address",
        url: "/forms/example/address-form.html",
        actions: [ "back", "finish" ]
      }
    ]
  }
}

Here the Form object has just the three required properties; id, url and actions.

Required Form Properties

id - a name which identifies the form, must be is unique in the current list.

url - the path to the form file on disk, relative to the root of the webapp.

actions - an array of Actions this form can perform. There are a few simple action strings; "next", "back" and "cancel", but they can be more complex. More on actions here: FormFlow > Action.

Optional Form Properties

docBase - a form-level docBase which overrides the flow docBase while this form or it's subforms are in use.
