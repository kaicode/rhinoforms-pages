FormFlow

The FormFlow is defined as a JavaScript file containing a single object declaration. In it's simplest form it looks like this: 

{
  docBase: "/rootNode",
  formLists: {
    main: [
      {
      	id: "name",
      	url: "/forms/example/name-form.html",
      	actions: [ "finish" ]
      }
    ]
  }
}

Here the flow object has just the two required properties; docBase and formLists.

Required FormFlow Properties

docBase - the base path in the DataDocument which the flow will act on.

formLists - a map of Form arrays. Each form in an array will be presented to the user in order as the 'next' and 'back' form actions are used. There must be a key named 'main', this is the entry point when a flow is loaded. Other form lists can be accessed using form Actions.

Optional FormFlow Properties

{
  docBase: "/myRootNode",
  defaultInitalData: "the-initial-data.xml",
  libraries: ["js/my-util-methods.js"],
  formLists: { ... }
}

defaultInitalData - the name of an XML file in your flow directory which will be used to create the DataDocument for each new session.

libraries - an array of relative paths to JavaScript libraries you would like to be loaded on the server for use in validationFunctions.
