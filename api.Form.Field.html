Rhinoforms Form Fields are plain HTML <input> fields:

<input name="firstName" type="text" />

Validation

Field validation rules are defined in the HTML against the input field. There are two ways to define rules;

  • Validation Keywords - cover the basics - required, email etc.
  • Validation Functions - more complex, can reference other fields.

When Validation Happens

By default validation happens when the form is submitted but this behaviour can be changed using the rf.validateOn attribute: 

<input name="age" type="text" rf.validation="required"
  rf.validateOn="inputBlur" />
At the moment "inputBlur" is the only available value. This will cause the field to be validated when it looses keyboard focus (usually when the next field is selected).

DataDocument Location

Fields are bound to a node in the DataDocument implicitly. The path to a field's DataDocument node is resolved using the current docBase and the input field name.

For example, a form with a docBase of "/myData/address" with the following fields:

<input name="line1" type="text" />
<input name="line2" type="text" />

Will produce the following nodes:

<myData>
  <address>
    <line1>10 Downing Street</line1>
    <line2>London</line2>
  </address>
</myData>

Nodes deeper than the current docBase can also be produced by using dot notation in the input field name. A form with a docBase of "/profile" and a field:

<input name="about.interests" type="text" />

Will produce nodes like:

<profile>
  <about>
    <interests>Some text</interests>
  </about>
</profile>
