Validation Keywords provide validation for a single field and are declared using rf.validation. 

<input name="firstName" type="text" rf.validation="required" />

Multiple keywords can be used separated by a space: 

<input name="email" type="text" rf.validation="required email" />

Some keywords take parameters which are given as a name/value map inside round brackets: 

<input name="year" type="text" rf.validation="date({ format: 'YYYY' })" />

Predefined Validation Keywords

required - some input must be given.

email - input must match an email format.

date - input must match given format and must be a valid date on the Gregorian calendar. Specify the date format using a settings map with 'format' key, see example. Defaults to "DD/MM/YYYY". Uses the Moment.js library, see full list of parsing formats. 

<input name="m" type="text" rf.validation="date({ format: 'MM/YY' })" />

between - input must be between two numbers. Should be used with an input mask to restrict input characters to numbers. The two numbers can be taken from the DataDocument using Value Injection 

<input name="pickANumber" type="text" 
  rf.validation="between({ decimalA: '1', decimalB: '100' })" />

pattern - input must match a given Regular Expression.
Parameters: regex (required), errorMessage (optional, overrides the default error message). 

<input name="price" type="text"
  rf.validation="pattern({ regex: '^[0-9]+\\.[0-9]{2}$', errorMessage: 'Please include two decimal places.' })" />
This example has a regular expression which requires one or more numbers followed by a decimal point and then exactly two numbers.

To add your own validation keywords see Extending Rhinoforms.
