The FormFlow is the declaration of a set of forms and their allowed behaviour.

The FormFlow is defined as a JavaScript file containing a single object declaration. Here is a simple flow with just two forms: 

{
  docBase: "/root",
  formLists: {
    main: [
      {
        id: "name",
        url: "/forms/example/name-form.html",
        actions: [ "next" ]
      },
      {
        id: "address",
        url: "/forms/example/address-form.html",
        actions: [ "back", "finish" ]
      }
    ]
  }
}

In bold are the two required properties; docBase and formLists.

Required FormFlow Properties

docBase - the base path in the DataDocument which the flow will act on.

formLists - a map of key/Form array pairs. There must be a key named 'main', this is the entry point when a flow is loaded. Other form lists are accessed using form Actions.

Optional FormFlow Properties

{
  docBase: "/myRootElement",
  defaultInitalData: "the-initial-data.xml",
  libraries: ["js/my-util-methods.js"],
  formLists: { ... }
}

defaultInitalData - the name of an XML file in your flow directory which will be used to create the DataDocument for each new session.

libraries - an array of relative paths to JavaScript libraries you would like to be loaded on the server for use in validationFunctions.
