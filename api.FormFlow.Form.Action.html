There can be one or many Actions in the Form.actions array. The single Action of the Form in the FormFlow below is in bold: 

{
  docBase: "/rootNode",
  formLists: {
    main: [
      {
      	id: "name",
      	url: "/forms/example/name-form.html",
      	actions: [ "finish" ]
      }
    ]
  }
}

A Form can only use an Action defined in it's actions list.

Actions can be bound to buttons and other HTML elements for the user to trigger.

There are three reserved action names: "next", "back" and "cancel".

"next" - validates the form, stopping for any errors, posts the field values to the server and loads the next form in the list.

"back" - posts the field values to the server without validating and loads the previous form in the list. The values of any fields that were valid will be persisted, any invalid or blank fields will be dropped.

"cancel" - loads the previous form in the list without validating or posting the field values.

As well as stepping forward and backward through the form list you can also jump to a form by using it's id....
