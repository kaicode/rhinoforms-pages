Concepts

  • Forms
  • DataDocument
  • FormFlow

Forms

Forms are written in HTML using familiar form and input elements, with some extra markup for validation rules etc. Form validation is supported by the Rhinoforms JavaScript library in the browser and repeated server-side upon form submission.

The magic happens when the forms are loaded through the Rhinoforms Servlet, this reads the form HTML and uses the DataDocument to inject any existing values into your input elements and any other placeholders. At this stage drop-down options and some other content can be generated. The Rhinoforms Servlet also records everything about your form so that complete validation can be done when the form is submitted.

DataDocument

This is the XML Document which stores captured form data. It can start as just a root element and will be built up automatically as each form is submitted.

The DataDocument can be pre-populated using template file, or the result of a web-service call. This way Rhinoforms can be used to manipulate existing XML entities.

FormFlow

The FormFlow is the model. This includes what sets of forms are available in the form session, what part of the DataDocument each form targets and how the forms interact with each other and external web-services.

The FormFlow definition is written in JavaScript and can be as simple or as complex as you need. The FormFlow definition is used by the Rhinoforms Servlet; all the heavy lifting happens there.
