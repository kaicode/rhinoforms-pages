Rhinoforms Forms are written in good old HTML.

One of the main principles of Rhinoforms is that forms code should be readable. Anyone with basic HTML knowledge should be able to read and understand the code, not take one look and run half a mile in the other direction (like XForms, cough!)

A simple form might look like this:

<form rhinoforms="true">
  <input name="firstName" type="text" rf.validation="required" />
  <input name="email" type="text" rf.validation="required email" />
  <input type="submit" action="next" />
</form>

A Rhinoforms Form must be marked with the attribute rhinoforms='true', this tells the parser to process the form.

There should be no action or method attributes on the <form> tag. This is because all forms are posted to the Rhinoforms Servlet for server-side validation. The action url does not make it's way into the browser, this way no one can bypass the forms. The collected data can make it's way to a web endpoint using a submission, or the Rhinoforms Servlet may just respond with the next form. All that logic is tucked away in the FormFlow.

The <input> fields have some custom attributes on them:

<input name="firstName" type="text" rf.validation="required" />
<input name="email" type="text" rf.validation="required email" />
<input type="submit" action="next" />
