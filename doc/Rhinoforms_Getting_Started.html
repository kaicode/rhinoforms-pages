Rhinoforms: Getting Started

Concepts

  • DataDocument
  • FormFlow
  • Forms

DataDocument

This is the XML Document which stores captured form data. It can start as just an empty root node and will be built up automatically as each form is submitted.

The DataDocument can be pre-populated using a static template XML file, or the result of a web-service call. This way Rhinoforms can be used for editing existing entities.

FormFlow

A FormFlow is a unit of forms and logic. The FormFlow models what forms are in the set, what part of the DataDocument each form targets and how the forms interact with each other and external web-services.

The FormFlow definition is written as a JavaScript object and can be as simple or as complex as you need. It is only used server-side as all the heavy lifting happens there.

Forms

Forms are written as plain HTML using form and input elements with some extra markup to record validation rules etc. Forms are loaded via the Rhinoforms Servlet which parses and modifies the form HTML before they get to the browser. Existing values from the DataDocument are pushed into the form inputs and any other placeholders. Drop-down options and other content can also be generated. Form validation is supported by the Rhinoforms JavaScript library in the browser and repeated server-side upon form submission.
