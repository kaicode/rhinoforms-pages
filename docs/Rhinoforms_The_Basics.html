Rhinoforms: The Basics

Who is Rhinoforms for?

Rhinoforms is for anyone with basic HTML and JavaScript skills who wants to capture user input.

What can Rhinoforms do?

Rhinoforms captures form data as an XML document.

Form validation can be provided based on pre-defined keywords (required, email, pattern, etc) or short JavaScript functions referencing other fields.

Validation is run in the browser for a quick response time, and for bullet-proof security validation is run again on the server.

Rhinoforms can inject values from the XML document into any part of a form, not just the input fields.

Rhinoforms can post to a web-service and merge the response back into the XML document, providing XSLT transforms along the way if required.

What does Rhinoforms markup look like?

The forms code is based on plain old html forms. Here is an example: 

<form rhinoforms="true">
  <input name="firstName" type="text" rf.validation="required" />
  <input name="email" type="text" rf.validation="required email" />
  <input type="submit" action="next" />
</form>
You will probably want to add labels.

You will also need a form-flow definition file, writen in JavaScript, listing what forms are in the set, what actions they have and what part of the XML document each form is focused on. Fields are bound to a path and node-name in the XML document based on the focus path of the form and the field name.
